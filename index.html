Social Platform

UX/UI Responsive Web Design

To get things started, I want to show off my version of a social profile platform that provides responsive desktop, tablet, and mobile design.

Before taking off into developing, I start with a pencil and graph paper. I use Atomic Design Principles in which I start figuring out the atoms of the composition. I nest, frame and layout the navigation bar, main components, and the side bars.

This specific project focuses more on the main composition and two side bars.

I usually stray away from pixels and points to attain a smooth flow throughout the experience. Usually, I differ between percentage units and viewport units.

If you play with the size of your screen then you can see the difference.

The three main layouts are desktop, tablet, and mobile. Due to people using different devices, the breakpoints are always visually beneficial, but also less stressful and eye-straining.

Social Platform introduces 1 column and up to 3 columns depending on viewport.

Wireframes

Before going into Visual Studio Code to type out the coding, I designed the layout with wireframes using Figma. For a responsive website, the layout consisted of a mobile and desktop version. I took into account the dimension differences- especially with people using Apple, Android, Chrome, etc..

As a designer, it is my job to focus on information structures, controlled and uncontrolled metadata, and findability. I re-created the mouse cursor to function with text, images, and on-click images/text but also designed them with the website's aesthetics in mind.

The goal is to exercise a user-friendly platform at every step of the way. As one of the tools to access user data, I use Hotjar and Google Analytics that provides information on where exactly people click, hover, and interact with elements on my webpages. This information allows me to gather passive feedback from user's click data and refine my website.

I have used an accumulation of Javascript, CSS, and SASS to accent the HTML. In consideration to ongoing coders, I have organized everything into seperate files and labeled everything as I have developed the code.
